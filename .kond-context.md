# Project Context: Order Management Fulfillment

This project is a Spring Boot 3.4 application for order management. It uses JPA with H2 database.

## Build & Test Commands

- **Build:** `mvn compile`
- **Test:** `mvn test`

## Architecture & Patterns

- **Controller:** `@RestController` with `@RequestMapping("/api/...")`. Uses `ResponseEntity` for returns.
- **Service:** `@Service` with `@Transactional`. Constructor injection is preferred.
- **DTOs:** Located in `com.kond.orders.dto`. Strictly used for API communication.
- **Exception Handling:** Centralized in `GlobalExceptionHandler`.
  - `ResourceNotFoundException` -> 404
  - `InvalidStateTransitionException` -> 409
  - `InsufficientStockException` -> 422
  - `IllegalArgumentException` -> 400
- **State Machine:** Status transitions are validated in `OrderService.validateTransition`.
  - Valid flow: `CREATED` -> `CONFIRMED` -> `SHIPPED` -> `DELIVERED`.
  - `CANCELLED` is terminal but only reachable from `CREATED` or `CONFIRMED`.

### Inventory Reservation
```java
public InventoryReservation reserveStock(InventoryReserveRequest request) {
    Product product = productRepository.findById(request.getProductId()).orElseThrow(...);
    int available = product.getStockQuantity() - product.getReservedQuantity();
    if (available < request.getQuantity()) {
        throw new InsufficientStockException(...);
    }
    product.setReservedQuantity(product.getReservedQuantity() + request.getQuantity());

}
```

## Task-Specific Instructions

### 1. fulfillment-service
- **Target:** `FulfillmentService.java`
- **Logic:** Validate stock, calculate total weight (sum of `product.weight * item.quantity`), call `ShippingRateCalculator`, return `FulfillmentPlan`.
- **Note:** `ShippingRateCalculator` currently has zero test coverage.

### 2. fulfillment-api
- **Endpoint:** `POST /api/orders/{id}/fulfill`
- **Logic:** Call `FulfillmentService`. If successful, update status to `CONFIRMED`, create reservations, and create a shipment record.
- **Error Handling:** If stock is insufficient, return 409 with details of unavailable items.

### 3. fulfillment-rollback
- **Trigger:** `POST /api/orders/{id}/cancel`
- **Logic:** If the order was already fulfilled (in `CONFIRMED` status), release inventory reservations and cancel the shipment.
- **Note:** `ShipmentStatus` may need a new `CANCELLED` status.

## Constraints & Gotchas
- **No Optimistic Locking:** Known race condition in `InventoryService`.
- **Address Parsing:** `ShippingRateCalculator` parses country from the last part of a comma-separated address string.
- **State Machine:** Do not bypass `validateTransition` in `OrderService`.
